Lessons 1–2

01
Brand Context
สร้าง CLAUDE.md — briefing ถาวรสำหรับทุก session
ตั้งค่า OpenRouter plugin — gateway สำหรับ AI models
สร้าง brand-guidelines.md ด้วย 6 คำถามกุญแจ
สร้างคำสั่ง /brand
Output: CLAUDE.md, brand/brand-guidelines.md, /brand command
02
Researcher Agent
สร้าง SKILL.md สำหรับ content-researcher
ค้นหา trending topics ล่าสุด (30 วัน)
วิเคราะห์ competitor content และ audience questions
สร้างคำสั่ง /research
Output: .claude/skills/content-researcher/SKILL.md, research/ folder
ทุกบทเรียนผลิตไฟล์จริง — ไม่ใช่แค่ความรู้ที่หายไปหลังปิดหน้าต่าง

Lessons 3–4

03
Content Calendar
ตั้งค่า NocoDB MCP — ต่อ Claude กับ database
สร้าง Content Calendar table (7 fields)
Generate 20–30 รายการสำหรับเดือนถัดไป
รายการกระจายหลาย platform และ format
Output: NocoDB table พร้อม 20+ entries ครอบทุก pillar
04
Content Writer
สร้าง SKILL.md สำหรับ content-writer
เขียน social post ในเสียงของแบรนด์
เขียน article 800–1500 คำ
เขียน video script ที่พูดได้จริง
Output: 3+ ไฟล์ใน content/ folder, NocoDB status = Drafted
Key: เนื้อหาต้องฟังดูเหมือนคุณ ไม่ใช่ generic AI marketing copy

Lessons 5–6

05
Content Designer
สร้าง SKILL.md สำหรับ content-designer
Generate รูปที่ตรงสีและสไตล์แบรนด์
ใช้ OpenRouter image generation (GPT-image model)
Aspect ratio ถูก platform: 1:1, 9:16, 16:9
Output: 2+ รูปใน designs/ folder, NocoDB status = Designed
06
Video Maker
ตั้งค่า HeyGen MCP — ต่อกับ AI avatar platform
เลือก avatar และ voice ที่เหมาะกับแบรนด์
สร้าง SKILL.md สำหรับ video-maker
Generate วิดีโอ (render 2–5 นาที)
Output: วิดีโอ AI avatar พูด script ของคุณ 1 ชิ้น

Lessons 7–8

07
Landing Page
ตั้งค่า Vercel MCP — ต่อกับ web hosting
Claude สร้าง landing page HTML + CSS ให้
ใช้สี typography และ content จาก brand guidelines
Deploy ขึ้น internet — ได้ URL *.vercel.app จริง
Output: landing page live บน internet ทดสอบบนมือถือได้
08
Publisher + Wrap-up
สร้าง content-publisher skill (format per platform)
สร้าง /content-engine master command
รัน full pipeline ครั้งแรก: research → plan → write → design → format
เรียนรู้ 3 แนวทางปรับใช้ต่อ
Output: /content-engine command พร้อมใช้งาน

ความจริงที่ควรรู้ก่อนเริ่ม

สิ่งที่ระบบทำได้ สิ่งที่ยังต้องทำเอง
วิจัย, วางแผน, เขียน, ออกแบบ, ทำวิดีโอ ตรวจและอนุมัติก่อนโพสต์
สร้าง landing page และ deploy โพสต์จริงลงแพลตฟอร์ม (copy-paste)
Format content สำหรับแต่ละ platform เชื่อมต่อโดยตรงกับ social media API
สร้าง content calendar ล่วงหน้า 1 เดือน วิเคราะห์ performance หลังโพสต์
รัน pipeline ทั้งหมดด้วยคำสั่งเดียว A/B testing และ analytics dashboard
ระบบนี้จัดการ production work — คุณจัดการ judgement calls
